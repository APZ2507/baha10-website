# אתר שירותי הבסיס — מדריך תחזוקה

## מבנה הקבצים

```
index.html      ← האתר עצמו. לא נוגעים בו לעדכון מידע שוטף.
data.json       ← כל המידע הדינמי: שעות, טלפונים, קו 10, שאטלים, נהלים. כאן עורכים.
header-bg.png   ← תמונת הרקע בשער העליון (אופציונלי).
*.mp4           ← סרטוני הדרכה (למשל Meditik.mp4, WeaponForm.mp4).
```

**חשוב:** `index.html` טוען את `data.json` בכל פעם שמישהו נכנס לאתר. כדי לעדכן שעות, טלפונים,
או כל מידע אחר — **עורכים רק את `data.json`**. אין צורך לגעת ב-`index.html` בכלל, ואין צורך
בשום ידע בתכנות. `data.json` הוא קובץ טקסט רגיל שאפשר לערוך גם ישירות באתר GitHub.

אם משום מה `data.json` לא נטען (לדוגמה תקלת רשת, או שהקובץ נמחק בטעות) — האתר לא נשאר ריק;
הוא נופל אוטומטית לברירת מחדל קבועה שכתובה בתוך `index.html`, כדי שהמידע הבסיסי תמיד יוצג.

## איך לערוך את data.json

פותחים את `data.json` בכל עורך טקסט (גם Notepad, גם ישירות בדפדפן ב-GitHub — לוחצים על העיפרון
לעריכה). מבנה הקובץ מחולק לסעיפים ברורים:

### 1. `services` — כל השירותים בבסיס (מרפאה, חדר אוכל, שק"ם וכו')

```json
"clinic": {
  "name": "מרפאה",
  "category": "health",
  "icon": "⚕",
  "hours": {
    "0": [{ "start": "08:00", "end": "16:00" }],
    "1": [{ "start": "08:00", "end": "16:00" }],
    "2": [], 
    "3": [],
    "4": [],
    "5": [],
    "6": []
  },
  "note": "טקסט חופשי שמופיע מתחת לשעות — אפשר גם קישור: <a href=\"...\">כאן</a>",
  "videoUrl": "Meditik.mp4",
  "videoCaption": "הדרכה: איך מזמינים תור",
  "waGroupUrl": "",
  "waGroupLabel": ""
}
```

- **המפתחות `"0"` עד `"6"`** הם ימי השבוע: 0=ראשון, 1=שני, 2=שלישי, 3=רביעי, 4=חמישי, 5=שישי, 6=שבת.
- כל יום מחזיק **רשימה** של חלונות שעות — כך אפשר לתמוך בכמה טווחים באותו יום (לדוגמה חדר אוכל
  עם ארוחת בוקר/צהריים/ערב). יום עם רשימה ריקה `[]` = סגור באותו יום.
- אפשר להוסיף `"label"` לכל חלון (לדוגמה `"label": "שחרית"` בבית הכנסת, או `"label": "בנים"` בחדר כושר).
- `note` הוא טקסט חופשי (תומך גם ב-HTML פשוט כמו קישורים).
- `videoUrl` — שם קובץ וידאו מקומי (חייב לשבת באותה תיקייה), או קישור embed מיוטיוב. השאירו `""` אם אין סרטון.
- `waGroupUrl` — קישור לקבוצת ווטסאפ (למשל עדכוני חדר כושר). השאירו `""` אם אין.

**להוספת שירות חדש:** מוסיפים בלוק חדש בתוך `services`, עם `id` ייחודי משלכם כמפתח (למשל `"barbershop"`),
ומקפידים על אותו מבנה בדיוק (name/category/icon/hours וכו'). `category` חייב להיות אחד מ:
`health`, `food`, `logistics`, `sport`, `transport`, `prayers`.

**למחיקת שירות:** פשוט מוחקים את הבלוק שלו מתוך `services` (כולל הפסיק שמפריד בינו לבין השכן).

### 2. `phones` — טלפונים חשובים

```json
{ "name": "מוקד רפואי", "role": "חירום ופניות דחופות", "number": "100" }
```
רשימה פשוטה. מוסיפים/מוחקים/עורכים שורות כרצונכם.

### 3. `busRoutes` + `busHours` — קו 10

```json
"busRoutes": [
  { "id": 1, "name": "מסלול 1", "minuteOffset": 0, "stops": ["רחבת היסעים", "ש\"ג/בינוי", "..."] }
],
"busHours": { "firstHour": 8, "lastHourNormal": 19, "lastHourThursday": 17 }
```
- `minuteOffset` = בכמה דקות אחרי כל שעה עגולה יוצא המסלול (0, 20, 40...).
- `stops` = רשימת התחנות לפי סדר הנסיעה.
- `firstHour`/`lastHourNormal`/`lastHourThursday` שולטים על טווח השעות הכללי של הקו.

### 4. `transportInfo` — עלון "דרכי הגעה ויציאה" (קו 392)
כותרת, פסקאות טקסט (`body` היא רשימה — כל איבר הוא פסקה נפרדת), סרטון וקישור הזמנה.

### 5. `shuttles` + `shuttleApps` — שאטלים מהערים
```json
{ "route": "תל אביב ← בת ים ← ראשון לציון", "days": "ראשון, חמישי", "whatsapp": "https://chat.whatsapp.com/..." }
```

### 6. `guidelines` — דגשים ונהלים
```json
{ "title": "אבטחת מידע", "points": ["נקודה ראשונה", "נקודה שנייה"] }
```
כל אובייקט הוא נושא נפרד עם כותרת ורשימת נקודות. מוסיפים/מוחקים נושאים או נקודות כרצונכם.

### 7. `absorptionSteps` — תהליך הקליטה
```json
{ "title": "רישום ואימות פרטים אישיים", "desc": "תיאור קצר של השלב" }
```
רשימה מסודרת של 6 השלבים המוצגים כציר זמן.

## חוקים חשובים בעריכה

1. **שומרים על תחביר JSON תקין**: כל מחרוזת בין מרכאות כפולות `"..."`, פסיק אחרי כל איבר חוץ
   מהאחרון ברשימה, סוגריים מסולסלים/מרובעים נסגרים כמו שנפתחו. שגיאת תחביר קטנה (פסיק חסר,
   מרכאה לא סגורה) תגרום לכל הקובץ לא להיטען — האתר יחזור אוטומטית לברירת המחדל הישנה, אבל
   כדאי לתקן מהר.
2. **לפני שמפרסמים שינוי**, מומלץ להדביק את תוכן הקובץ בכלי בדיקת JSON חינמי (לדוגמה
   jsonlint.com) כדי לוודא שאין שגיאת תחביר.
3. **גיבוי**: לפני עריכה משמעותית, שווה לשמור עותק של הקובץ הנוכחי בצד (Ctrl+A, Ctrl+C להעתקה
   למחברת, או פשוט מורידים את הקובץ הקיים).

2. שומרים ("Commit changes" ב-GitHub).
3. Vercel מזהה את השינוי אוטומטית ומפרסם גרסה חדשה תוך שניות עד דקה.
4. כל מי שנכנס לאתר מהרגע הזה — מכל מכשיר, בלי צורך לרענן שום דבר במיוחד — רואה את הגרסה המעודכנת.

אין צורך לפתוח VS Code, להריץ פקודות, או לגעת ב-`index.html` בשום שלב.
